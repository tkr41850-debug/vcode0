<objective>
Rewrite the SQLite migration runner to the CONTEXT-locked numbered-`.sql` shape, widen the `Store` port to be the single external persistence boundary (graph + runs + events + lifecycle), and leave the repo typecheck-clean. This closes Phase 2 Success Criterion 1 (typed Store port over all graph / run / milestone / summary / usage-rollup operations) and unblocks Phase 3 workers which consume `Store` exclusively.

Purpose: Persistence is the one hard contract every later phase depends on. Today the repo has a real `PersistentFeatureGraph` + `SqliteStore` + 6 TS migrations that don't match the CONTEXT lock (numbered `.sql`), and the `Store` port only covers agent_runs + events — graph access is a parallel class. This plan consolidates both into one port, swaps the migration shape, and closes the small schema gaps flagged by RESEARCH (merge-train SHA columns, `integration_state`) so Phase 6 lands without schema churn.

Output: One widened `Store` interface, two `.sql` migrations (consolidated 0001 baseline + 0002 merge-train executor state), a numbered-.sql `MigrationRunner`, and a typecheck-clean tree.
</objective>
