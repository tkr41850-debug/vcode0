<objective>
Replace the current top-level-planner stub with a real prompt-to-proposal flow. The composer must be able to start a top-level planner run from plain text, the agent must operate over the milestone/feature graph rather than a single feature, and the resulting proposal must still go through explicit approval/apply. The same plan also closes the missing proposal-surface gap discovered during research: current tools only support `add_milestone` plus feature/task CRUD, but Phase 7 needs milestone edits, feature moves between milestones, and feature split/merge semantics in the draft graph.

This plan is the skeleton of Phase 7. It does NOT implement continue-vs-fresh session choice (07-04), unified inbox resolution (07-02), or pause/respawn (07-03). It only ensures that a top-level planner run can exist, produce a proposal, and apply additive changes safely.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI composer -> top-level planner runtime | Raw user prompt text crosses into the planner agent; proposal apply must remain explicit so prompt text never mutates graph state directly |
| Top-level proposal -> authoritative graph | Draft proposal ops can touch milestone/feature topology; additive-only guardrails must prevent silent mutation of started/completed work |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | Top-level planner proposal apply | mitigate | Keep proposal-first flow; only scheduler approval applies graph mutations. |
| T-07-01-02 | Tampering | Started/completed work mutated by replan | mitigate | Extend stale/warning logic so non-additive top-level ops are skipped and surfaced. |
| T-07-01-03 | Elevation of Privilege | TUI plain-text path bypasses event queue | mitigate | Composer only enqueues top-planner dispatch; no direct graph mutation in UI code. |
| T-07-01-04 | Denial of Service | Missing planner run scope forces ad-hoc run ids / state leakage | mitigate | Add a persisted top-level planner run type and route everything through store-backed runs. |
</threat_model>
