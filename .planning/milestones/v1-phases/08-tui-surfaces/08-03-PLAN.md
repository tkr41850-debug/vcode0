<objective>
Complete the manual DAG edit slice of Phase 8 by surfacing the missing command-first
manual graph actions in the TUI composer: move feature, split feature, merge features,
and reorder tasks — while keeping the current add/edit/remove/reweight/cancel baseline,
reusing the existing proposal draft + approval flow, and preserving the docs-aligned
"manual edits win" model.

Today the lower layers already support most of the hard mutations:
- proposal/tooling already supports `moveFeature`, `splitFeature`, and `mergeFeatures`
- core graph already supports `reorderTasks(...)`
- the TUI already exposes add/edit/remove dependency work through `ComposerProposalController`

So the real 08-03 gap is not inventing new graph capabilities from zero. It is exposing the
missing operator surface cleanly and honestly from the existing TUI composer shell.
</objective>
