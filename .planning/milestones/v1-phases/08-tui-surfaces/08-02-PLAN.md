<objective>
Ship the next real Phase 8 operator surface: a merge-train overlay that makes the
already-shipped Phase 6 queue state visible from the TUI and exposes one minimal
manual-priority control without widening into transcript UI, richer queue editing,
or general graph-editing polish.

Before this slice, merge-train state existed durably on feature rows and already
drove integration order plus cap parking, but the TUI exposed it only indirectly
through feature collab badges and inbox cap items. This plan adds a direct queue
surface while staying derived-state based and command-first like the inbox slice.
</objective>
