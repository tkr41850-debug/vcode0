## `replanning` disposition

The 01-01 executor flagged `replanning` for a canonical-docs decision.

**Decision:** exclude `replanning` from the composite validity matrix;
document the exclusion.

Rationale:

- The matrix in `state-axes.md` uses the same 10 WorkControl values as
  `test/unit/core/fsm/composite-invariants.test.ts` `WORK_VALUES`. Adding
  `replanning` would desynchronise the doc from the test and false-positive
  the Phase 11 drift check.
- `replanning` is a transient repair-escalation state with the same
  cross-axis constraints as `planning` (Rule 7: pre-branch phase requires
  collab ∈ {none, cancelled}) plus Rule 4 (cancelled collab freezes it).
  Rather than duplicate 42 rows that restate Rule 7, `state-axes.md`
  explicitly documents the exclusion with a pointer to the axis test that
  carries ground truth for `replanning` transitions.

The exclusion rationale is captured inline in `state-axes.md § Note on
replanning`.

## Matrix fidelity

The composite validity matrix in `state-axes.md` was generated by the same
rule logic as `isLegalByRules` in
`test/unit/core/fsm/composite-invariants.test.ts`. Row totals match:

- 420 total rows (10 × 7 × 6)
- 186 legal
- 234 illegal

Each illegal row includes the first-firing rule (by evaluation order) in the
`reason-if-illegal` column. Pipe characters in rule text (`collab=none|cancelled`,
`branch_open|merge_queued|integrating|conflict`, etc.) are backslash-escaped
so GitHub's Markdown renderer keeps the 5-column table shape.

Row iteration order (work outer → collab → run inner) matches the nested-loop
order in the test, so a Phase 11 doc-vs-code drift check can line-compare
directly.

## Prose-vs-table conflicts (flagged, not resolved)

Per the plan, prose that disagrees with the new decision tables is flagged
here rather than silently edited.

1. **`docs/operations/conflict-coordination.md § Mechanical Handling` step 3**
   says "Suspend the lower-priority task in that feature." The table in
   `coordination-rules.md § Suspend — same-feature overlap` is stricter:
   `run_state` stays `retry_await` while collab moves to `suspended` with a
   named `suspend_reason`. The narrative is still broadly correct but should
   be tightened to cite the `suspend_reason` enum values (`same_feature_path_lock`
   and `rebase_in_progress`). Recommend a follow-up docs plan to reconcile.
2. **`docs/operations/conflict-coordination.md § Cross-Feature Overlap`
   step 7** describes warning timing as "stays blocked for too long
   (`Date.now() - task.suspendedAt > threshold`)". The new table names the
   rule-owner as `core/warnings/` but the operations prose does not
   cross-link to the warning-rule module. No semantic disagreement; cross-link
   hygiene only.
3. The re-entry cap default value (10) appears in `coordination-rules.md`
   but is not currently stated in
   `docs/operations/conflict-coordination.md` or
   `docs/operations/verification-and-recovery.md`. If the cap value is
   configurable per-profile (as the table claims), the
   `budget-and-model-routing.md` page should own the default-value source of
   truth. Flagged for a future docs plan.

None of the above are semantic disagreements that change what the code must
do — they are narrative-vs-table tightenings. Leaving the narrative alone
per plan instructions.

## Deviations from Plan

### Auto-fixed issues

**1. [Rule 1 — Bug] Markdown-table pipe escaping in the composite matrix**
- **Found during:** Task 2 self-check (`grep` confirmed unescaped `|`
  inside reason-column text).
- **Issue:** Generated rule-reason strings contained literal `|` (e.g.
  `collab=none|cancelled`) which would break the 5-column table when
  rendered by GitHub Markdown.
- **Fix:** Post-process the matrix file to escape `|` to `\|` inside the
  reason cell only. Column separators untouched.
- **Files modified:** `docs/foundations/state-axes.md`.
- **Commit:** folded into `7f5bb2c`.
