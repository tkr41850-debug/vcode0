<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| test runner → pseudo-terminal shell | Test code submits shell text to a PTY that launches npm scripts. |
| pseudo-terminal shell → `src/main.ts --cwd` | Workspace path crosses from test fixture into production CLI cwd handling. |
| TUI composer → slash-command parser | Test-provided command strings exercise production slash-command parsing and graph mutation paths. |
| temp workspace → persistent `.gvc0/state.db` | TUI writes persistent state under a test-created workspace. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-12-02-01 | Tampering | `startTui()` shell command in TUI tests | mitigate | Keep the existing `shellQuote()` helper when interpolating temp workspace paths into `terminal.submit()`; never concatenate unquoted paths. |
| T-12-02-02 | Tampering | `.gvc0/state.db` test fixture state | mitigate | Use a fresh temp workspace per test and preserve `test.afterEach` cleanup with `terminal.kill()` plus recursive `fs.rm`. |
| T-12-02-03 | Information Disclosure / Denial of Service | Planner/provider execution from PTY | mitigate | Use local slash commands and deterministic graph/draft flows only; do not call live LLM providers or run full autonomous workers in the TUI smoke. |
| T-12-02-04 | Repudiation | 12-03 traceability handoff | mitigate | Name the test with `golden path tui e2e smoke` and record exact commands/results in `.planning/phases/12-integration-polish/12-02-SUMMARY.md`. |
| T-12-02-05 | Denial of Service | `@microsoft/tui-test` workerpool `SIGSEGV` | mitigate | Stabilize/isolate the runner before adding coverage; verify with `npm run test:tui:e2e` before and after golden-path changes. |
</threat_model>
