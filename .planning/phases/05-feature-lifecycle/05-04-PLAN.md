<objective>
Close the two primary Phase 5 gaps — (SC4) executing_repair loop for verify failures and (SC5) hallucinated-progress rejection — and wire them into the end-to-end lifecycle.

Four concrete deliverables:
1. **Fix the verify→replan misroute.** `OrchestratorFeatures.completePhase('verify')` today routes `ok===false` directly to `replanning`, bypassing `executing_repair`. Rewire through a new `enqueueVerifyRepairs(featureId, issues[])` helper that fans out blocking/concern issues into one repair task each, counts the verdict as ONE repair attempt (RESEARCH O4), and preserves the existing cap-escalation to `replanning`.
2. **Add the `trailerObservedAt` audit trail.** New nullable column on `agent_runs`, symmetric setter/getter on `OrchestratorPorts`, set from the `commit_done` handler when `trailerOk===true` (RESEARCH O2).
3. **Gate `task_complete` on trailer observation.** Pre-check in the scheduler `result` handler: no trailer-OK commit → emit `task_completion_rejected_no_commit`, transition task to `failed`, feed `RetryPolicy` via a new `no_commit` failure kind (RESEARCH O3 — retry-in-pool, NOT straight-to-inbox).
4. **Update the stale assertion + add end-to-end coverage.** `feature-phase-agent-flow.test.ts:642-697` flips (RESEARCH SC4-G4). New E2E walks verify-fail → 1 repair task → executor passes → verify re-runs → pass.

Purpose: Satisfies SC4 + SC5 — the only two remaining Phase 5 success criteria not covered by 05-01/02/03. These are the primary gaps identified in RESEARCH.

Output:
- Orchestrator helper + migration + port/store/codec changes + event + retry-policy hook.
- Unit tests for the two pure paths (verify-repair fan-out, commit gate).
- Updated stale assertion + new E2E integration case.
</objective>
