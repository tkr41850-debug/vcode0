## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docs/test file paths -> filesystem reads | Drift tests read repository Markdown/source files and must stay constrained to fixed repo-local paths. |
| Source constants/guards -> canonical docs | Executable state/config behavior becomes documentation truth; tests must not duplicate stale logic or bless aspirational docs. |
| Canonical docs -> operator understanding | State/flow/coordination docs guide future implementation and diagnostics; stale claims can cause wrong recovery or merge-train expectations. |
| Reference docs -> user commands | TUI/CLI reference tells users which commands/surfaces exist; omissions hide shipped diagnostics. |

| T-11-02-03 | Repudiation | state/coordination docs | mitigate | link claims to executable sources (`src/core/fsm/index.ts`, `src/core/merge-train/index.ts`, `src/config/verification-layer.ts`) and protect key claims with Vitest |
| T-11-02-04 | Integrity drift | canonical documentation | mitigate | make `npm run check` fail on protected state-axis, execution-flow, coordination, data-model, verification, and command-inventory drift |

</threat_model>
