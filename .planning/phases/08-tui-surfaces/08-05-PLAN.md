<objective>
Close the final open Phase 8 slice by shipping a real TUI config editor surface together with
three distinct visible cancel levers.

This slice must finish Phase 8 honestly, which means it cannot stop at UI affordances. The repo
already has the overlay/command shell, but config is still boot-loaded once, workerCap is ignored,
featurePlanner/verifier model assignments are not distinctly consumed, and cancellation only exists
as a single feature-level graph/run abort path.

08-05 therefore needs to land as one coherent control-plane slice:
- persistent and validated config edits
- explicit live-apply hooks for hot settings
- distinct per-role model wiring
- explicit task-preserve / task-clean / feature-abandon semantics
- command-first TUI surface over those real behaviors
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI command input → config update path | user-provided command values become persisted runtime config |
| TUI cancel actions → git/worktree cleanup | operator-visible commands can destroy worktree/branch state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05-01 | Tampering | config persistence | mitigate | validate every update through `GvcConfigSchema` before apply/write; reject partial invalid state |
| T-08-05-02 | Denial of Service | worker cap / retry config | mitigate | clamp worker cap and numeric values to schema-valid positive ints; use explicit setters rather than reconstructing runtime |
| T-08-05-03 | Destruction | feature-abandon cleanup | mitigate | keep branch/worktree deletion in a dedicated helper with focused tests; use the preserve/clean/abandon split so destructive behavior is explicit in command name and notice text |
| T-08-05-04 | Integrity drift | role-model config UI | mitigate | wire each role field to a real runtime consumer and cover with unit tests so the menu cannot advertise dead settings |
</threat_model>
