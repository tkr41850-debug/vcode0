Idempotency: The tick loop is serial (one `tick()` at a time, not re-entrant).
The runner's guard is simple: `if (feature.collabControl !== 'integrating') return`.
Only one feature can have `collabControl === 'integrating'` (enforced by
`MergeTrainCoordinator.beginIntegration`). No additional concurrency guard is needed.

Integration runner shell outline:
```typescript
export async function runIntegrationIfPending(params): Promise<void> {
  // 1. Find integrating feature
  let feature: Feature | undefined;
  for (const f of params.graph.features.values()) {
    if (f.collabControl === 'integrating') { feature = f; break; }
  }
  if (feature === undefined) return;

  const featureDir = path.resolve(
    params.cwd ?? process.cwd(),
    worktreePath(feature.featureBranch),
  );

  // 2. Rebase onto main
  const rebase = await rebaseGitDir(featureDir, 'main');
  if (rebase.kind !== 'clean') {
    const error = rebase.kind === 'blocked'
      ? 'worktree missing during integration rebase'
      : `rebase conflict: ${rebase.conflictedFiles.join(', ')}`;
    await params.handleEvent({ type: 'feature_integration_failed', featureId: feature.id, error });
    return;
  }

  // 3. Shell verification
  const shellResult = await params.ports.verification.verifyFeature(feature);
  if (!shellResult.ok) {
    await params.handleEvent({
      type: 'feature_integration_failed',
      featureId: feature.id,
      error: shellResult.summary ?? 'merge-train shell verification failed',
    });
    return;
  }

  // 4. Agent review (REQ-MERGE-04)
  const run = { agentRunId: `run-integration:${feature.id}` };
  const agentResult = await params.ports.agents.verifyFeature(feature, run);
  if (!agentResult.ok) {
    await params.handleEvent({
      type: 'feature_integration_failed',
      featureId: feature.id,
      error: agentResult.summary ?? 'merge-train agent review failed',
    });
    return;
  }

  // 5. Fast-forward merge onto main
  try {
    const mainRepo = simpleGit(params.cwd ?? process.cwd());
    await mainRepo.merge([feature.featureBranch, '--ff-only']);
  } catch (err) {
    await params.handleEvent({
      type: 'feature_integration_failed',
      featureId: feature.id,
      error: `fast-forward merge failed: ${err instanceof Error ? err.message : String(err)}`,
    });
    return;
  }

  // 6. Emit complete
  await params.handleEvent({ type: 'feature_integration_complete', featureId: feature.id });
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| integration runner → git CLI (via simple-git) | Rebase and fast-forward merge operate on local disk; target is always `'main'` (hardcoded, not user-supplied). No injection surface. |
| integration runner → agent runtime (ports.agents) | Agent review returns a `VerificationSummary`; orchestrator uses only `ok` and `summary` fields. Agent cannot emit graph mutations from the verify context — toolset excludes graph mutators (enforced by Phase 5 tool host). |
| integration runner → verification service (ports.verification) | Shell commands come from `config.verification.mergeTrain.checks` (validated by zod schema at load time). No runtime user input reaches `runShell`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | `main` advanced on failed rebase or verify | mitigate | Fast-forward merge (step 5) is only reached after all of: rebase clean + shell verify ok + agent review ok. Any earlier failure exits with `feature_integration_failed` before `simpleGit.merge()` is called. |
| T-06-02-02 | Denial of Service | Integration runner re-enters on same tick | mitigate | Tick loop is serial; `tick()` is `async` and the loop `await`s each tick. Only one `runIntegrationIfPending` call can be active at a time. The `collabControl === 'integrating'` guard plus serial tick prevents double-run. |
| T-06-02-03 | Spoofing | Agent run ID collision with feature-phase verify | mitigate | Run ID prefix `run-integration:${featureId}` is distinct from `run-feature:${featureId}:verify`. Phase 5 `ensureFeaturePhaseRun` looks up by exact ID; no collision possible. |
| T-06-02-04 | Elevation of Privilege | Agent review triggers graph mutations | accept | Verify agent toolset (Phase 5 tool host) does not include planner tools or graph mutators. Agent can only call `submitVerify`, `raiseIssue`, and file-read tools. Accepted: enforced upstream by tool schema, not duplicated here. |
| T-06-02-05 | Denial of Service | Rebase hangs indefinitely on unresolvable conflict | accept | `rebaseGitDir` uses `simpleGit.rebase()` which returns promptly on conflict (does not wait for user input). Git non-interactive mode is the default in simple-git. Timeout enforcement deferred to Phase 9 crash recovery. |
</threat_model>
